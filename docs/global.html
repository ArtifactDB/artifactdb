globalRequestHeaders

Source:
  • globalRequestHeaders.js, line 6

Headers to be attached to all HTTP requests using the default functions (typically based on fetch). Keys are the header names while values are the header contents. This is mainly used to set the Authorization header for authentication.

Get permissions for a project.

Parameters:
Name Type Attributes Default Description
baseUrl string

Base URL of the REST API.
project string

Name of the project.
options object <optional>
 {}

Optional parameters.

Properties
Name Type Attributes Default Description
getFun function <optional>
<nullable>
 null

Function that accepts a single string containing a URL and returns a Response object. Defaults to the in-built fetch function with globalRequestHeaders.
Returns:

The permissions of the project. This is guaranteed to contain:

  • scope: string specifying the scope of the permissions, either "project" or "version".
  • read_access: string specifying who is allowed to read the project. This should be one of "public", "viewers", "owners" or "none".
  • write_access: string specifying who is allowed to update the project. This should be one of "owners" or "none".
  • viewers: array of strings specifying the users with read permissions.
  • owners: array of strings specifying the users with write permissions.
